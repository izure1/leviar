# Transform (변환) 가이드

안녕하세요! **Transform**은 객체가 월드 상의 **어디에, 어떤 방향으로, 어떤 크기로** 놓일지를 결정하는 가장 중요한 물리적 배치 정보입니다. 레비아 엔진은 "부모-자식(Parent-Child)" 계층 구조를 지원하여, 복잡한 3D 변환을 직관적으로 관리할 수 있도록 돕습니다.

---

## 📍 1. 변환 요소 (Components)

모든 객체는 `transform` 속성 아래에 다음과 같은 3축(x, y, z) 데이터를 가집니다.

-  **position: { x, y, z }** (px 단위)
   객체의 중심 위치입니다. (기본값: `0, 0, 0`)
   - 💡 Z축은 카메라와의 거리를 의미하며, 값이 커질수록 멀어집니다.
-  **rotation: { x, y, z }** (도 단위)
   객체의 각 축 기준 회전 각도입니다.
   - **회전 순서**: 월드 매트릭스 계산 시 `Z축 -> Y축 -> X축` 순서로 회전이 적용됩니다.
-  **scale: { x, y, z }** (배율)
   객체의 크기 배율입니다. (기본값: `1, 1, 1`)
-  **pivot: { x, y }** (0 ~ 1.0)
   변환의 중심(기준점)입니다. (기본값: `0.5, 0.5` - 정중앙)
   - `0, 0`: 좌측 상단
   - `1, 1`: 우측 하단

---

## ⛓️ 2. 계층 구조와 상속 (Hierarchy)

레비아 엔진은 객체 간의 부모-자식 관계를 통해 복잡한 움직임을 단순화합니다.

-  **addChild(child: LveObject): this**
   메서드를 통해 자식을 등록하면, 자식은 부모의 모든 변환 정보를 **상속**받습니다.
   - 💡 만약 부모가 회전하면, 자식도 부모를 중심으로 함께 공전합니다.
   - 💡 만약 부모의 크기가 커지면, 자식의 상대적인 위치와 크기도 함께 커집니다.
-  **Matrix World (월드 매트릭스)**
   내부적으로 부모의 월드 변환에 자신의 로컬 변환을 곱하여 최종 좌표를 계산합니다. 개발자는 복잡한 행렬 계산 없이 `position` 수치만 조절하여 직관적으로 배치할 수 있습니다.

---

## 👣 3. 추적 시스템 (Following System)

자식 관계(`addChild`) 외에도, 레비아 엔진은 **독립성을 유지하면서 따라다니는** 고유한 추적 시스템을 제공합니다.

-  **follow(target: LveObject, offset?: { x, y, z }): this**
   타겟 객체의 **위치(position)**만 실시간으로 복제하여 따라옵니다.
   - 💡 **자식 관계와의 차이점**: 타겟(부모)이 회전하거나 크기가 변해도 따라다니는 객체는 자신의 회전과 크기를 그대로 유지합니다. 이는 캐릭터 위에 떠 있는 이름표나 장식품을 구현할 때 매우 유용합니다.

---

## 💻 사용 예시

### 행성과 위성 만들기 (계층 구조 활용)
```typescript
const planet = world.createCircle({ style: { width: 100, color: '#e67e22' } });
const satellite = world.createCircle({ style: { width: 30, color: '#95a5a6' } });

// 자식으로 등록하면 부모(행성)를 따라 움직입니다.
planet.addChild(satellite);

// 위성을 옆으로 살짝 떼어 놓습니다.
satellite.transform.position.x = 150;

// 행성을 회전시키면 위성이 행성 주위를 공전합니다!
planet.animate({
  transform: { rotation: { z: '+=360' } }
}, 5000, 'linear');
```
